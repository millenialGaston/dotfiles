<span class="org-level-1">* Introduction</span>

  This Tutorial describes one of many ways to publishing Org-mode files to
  XHTML. We use the publishing mechanism to keep the <span class="org-verbatim">=*.html=</span> files separated
  from our <span class="org-verbatim">=*.org=</span> files and to access them via web browser. Simply exporting the
  Org-mode files to HTML would leave them in <span class="org-verbatim">=~/org/=</span>.

  The XHTML files we create will work every where, on any host, with or without
  network access, and even when accessed through the <span class="org-verbatim">=</span><span class="org-link"><span class="org-verbatim"><a href="file:///">file:///</a></span></span><span class="org-verbatim">=</span> protocol. To
  achieve this goal, we use

  - no absolute paths in HTML,
  - no server side scripting to navigate our output directories,
  - no <span class="org-verbatim">=base=</span> element (which is optional as of XHTML 1.0 strict) and
  - no software, but emacs, Org-mode and a web browser.

  All this means, we are able to check and use the result of work immediately,
  everywhere.

<span class="org-level-1">* Basics</span>

  Throughout this tutorial, let's assume that all our Org-mode files live beneath
  <span class="org-verbatim">=~/org/=</span> and we publish them to <span class="org-verbatim">=~/public_html/=</span>.

  Let's further assume you're already familiar with the note taking basics in
  org and able to add a little content to the Org-mode files we add to our project
  during this tutorial. Please add at least one headline to each of the files.

  The first thing to do is to create the folder <span class="org-verbatim">=~/org=</span>. This is where our notes
  files live. Inside <span class="org-verbatim">=~/org/=</span> we have a folder <span class="org-verbatim">=css/=</span> for stylesheets and
  scripts, and a folder called <span class="org-verbatim">=img/=</span> (guess what's in there).

  The first file we add to that folder (and to subdirectories later on) is called
  <span class="org-verbatim">=index.org=</span>. This name was choosen, since Org will publish the files to those
  with the basename of the source file plus the extension <span class="org-verbatim">=.html=</span> <span class="org-footnote">[fn:1]</span>. Thus
  <span class="org-verbatim">=~/org/index.org=</span> will once be <span class="org-verbatim">=~/public_html/index.html=</span> -- the startpage.

  Let's add another file and call it <span class="org-verbatim">=~/org/remember.org=</span>. After adding a
  stylesheet, <span class="org-verbatim">=~/org/=</span> looks like this:

  <span class="org-code">: ~/org/
</span>  <span class="org-code">:    |- css/
</span>  <span class="org-code">:    |  `- stylesheet.css
</span>  <span class="org-code">:    |- img/
</span>  <span class="org-code">:    |- index.org
</span>  <span class="org-code">:    `- remember.org
</span>
  When ever you want to add link to a file, do it the usual way. To link from
  <span class="org-verbatim">=index.org=</span> to <span class="org-verbatim">=remember.org=</span>, just write
  <span class="org-code">: <a href="file:remember.org">[[file:remember.org][remember]]</a>
</span>  Org will convert those links to HTML links for you on export:
  <span class="org-code">: &lt;a href="./remember.html"&gt;remember&lt;/a&gt;
</span>
  Same is true for images. To add an image, put it in <span class="org-verbatim">=~/org/img/test.jpg=</span> and
  refer to it by
  <span class="org-code">: <a href="file:img/test.jpg">[[file:img/test.jpg]]</a>
</span>
  You may test your links by clicking on them. To test image links you may also
  try to turn on <span class="org-verbatim">=iimage-mode=</span> <span class="org-footnote">[fn:2]</span> which works without problems here.

  Optionally, set up the stylesheet as shown in section <span class="org-link">Special comment
  section</span>. The recommended way is to use a real stylesheet though.

<span class="org-level-1">* Publishing the </span><span class="org-level-1"><span class="italic">/org/</span></span><span class="org-level-1"> project</span>

  To publish the <span class="org-verbatim">=~/org/=</span> project to HTML, we'll have to setup the variable
  <span class="org-verbatim">=org-publish-project-alist=</span> <span class="org-footnote">[fn:3]</span>.  I tend to split each project in three basic
  <span class="italic">/components/</span> as described in the manual. We need these different components,
  since we want org to use two different functions to publish dynamic content
  (org =&gt; html) and static content like scripts, images, stylesheets or even
  .htaccess files (org =&gt; copy). The third component is just for convenience and
  tells org to execute the former ones.

  <span class="org-verbatim">=org-publish-project-alist=</span> may be adjusted using customize (<span class="org-verbatim">=M-x
  customize-variable RET org-publish-project-alist RET=</span>), but I prefere to
  use an extra file to setup my projects, since there are quite a few. To follow
  this tutorial use the <span class="org-verbatim">=*scratch*=</span> buffer and put all the Lisp in this section
  in there.

  First of all, enter this into the <span class="org-verbatim">=*scratch*=</span> buffer:

<span class="org-block-begin-line">#+begin_src emacs-lisp
</span><span class="org-block">(</span><span class="org-block"><span class="keyword">require</span></span><span class="org-block"> '</span><span class="org-block"><span class="constant">ox-publish</span></span><span class="org-block">)
(</span><span class="org-block"><span class="keyword">setq</span></span><span class="org-block"> org-publish-project-alist
      '(

       </span><span class="org-block"><span class="comment-delimiter">;; </span></span><span class="org-block"><span class="comment">... add all the components here (see below)...
</span></span><span class="org-block">
      ))
</span><span class="org-block-end-line">#+end_src
</span>
  Be sure to put all the <span class="italic">/components/</span> below right there where the comment line
  is now.

<span class="org-hide">*</span><span class="org-level-2">* The </span><span class="org-level-2"><span class="italic">/notes/</span></span><span class="org-level-2"> component</span>

   The <span class="italic">/notes/</span> component publishes all the Org-mode files to HTML. Hence the
   <span class="org-verbatim">=publishing-function=</span> is set to <span class="org-verbatim">=org-publish-org-to-html=</span>. Here is an example
   of the notes component:

<span class="org-block-begin-line">#+begin_src emacs-lisp
</span><span class="org-block">(</span><span class="org-block"><span class="string">"org-notes"</span></span><span class="org-block">
 </span><span class="org-block"><span class="builtin">:base-directory</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"~/org/"</span></span><span class="org-block">
 </span><span class="org-block"><span class="builtin">:base-extension</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"org"</span></span><span class="org-block">
 </span><span class="org-block"><span class="builtin">:publishing-directory</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"~/public_html/"</span></span><span class="org-block">
 </span><span class="org-block"><span class="builtin">:recursive</span></span><span class="org-block"> t
 </span><span class="org-block"><span class="builtin">:publishing-function</span></span><span class="org-block"> org-html-publish-to-html
 </span><span class="org-block"><span class="builtin">:headline-levels</span></span><span class="org-block"> 4             </span><span class="org-block"><span class="comment-delimiter">; </span></span><span class="org-block"><span class="comment">Just the default for this project.
</span></span><span class="org-block"> </span><span class="org-block"><span class="builtin">:auto-preamble</span></span><span class="org-block"> t
 )
</span><span class="org-block-end-line">#+end_src
</span>
   Note, that <span class="org-verbatim">=headline-levels=</span> may be adjusted <span class="org-link"><a href="Overwrite the defaults">on a per file basis</a></span> to overwrite
   the default.

   The most important settings here are:

   <span class="org-table">| </span><span class="org-formula"><span class="org-verbatim">=base-directory=</span></span><span class="org-formula">       </span><span class="org-table">| The components root directory.                                                                                                |</span>
   <span class="org-table">| </span><span class="org-formula"><span class="org-verbatim">=base-extension=</span></span><span class="org-formula">       </span><span class="org-table">| Filename suffix without the dot.                                                                                              |</span>
   <span class="org-table">| </span><span class="org-formula"><span class="org-verbatim">=publishing-directory=</span></span><span class="org-formula"> </span><span class="org-table">| The base directory where all our files will be published.                                                                     |</span>
   <span class="org-table">| </span><span class="org-formula"><span class="org-verbatim">=recursive=</span></span><span class="org-formula">            </span><span class="org-table">| If </span><span class="org-table"><span class="org-verbatim">=t=</span></span><span class="org-table">, include subdirectories - we want that. Subdirectories in </span><span class="org-table"><span class="org-verbatim">=:publishing-directory=</span></span><span class="org-table"> are created if they don't yet exist. |</span>
   <span class="org-table">| </span><span class="org-formula"><span class="org-verbatim">=publishing-function=</span></span><span class="org-formula">  </span><span class="org-table">| If and how org should process the files in this component. In this case: convert the Org-mode files to HTML.                  |</span>

<span class="org-hide">*</span><span class="org-level-2">* The </span><span class="org-level-2"><span class="italic">/static/</span></span><span class="org-level-2"> component</span>

   The <span class="italic">/static/</span> component just copies files (and their folders) from
   <span class="org-verbatim">=:base-directory=</span> to <span class="org-verbatim">=:publishing-directory=</span> without changing them. Thus
   let's tell Org-mode to use the function <span class="org-verbatim">=org-publish-attachment=</span>:

<span class="org-block-begin-line">#+begin_src emacs-lisp
</span><span class="org-block">(</span><span class="org-block"><span class="string">"org-static"</span></span><span class="org-block">
 </span><span class="org-block"><span class="builtin">:base-directory</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"~/org/"</span></span><span class="org-block">
 </span><span class="org-block"><span class="builtin">:base-extension</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"css</span></span><span class="org-block"><span class="string"><span class="regexp-grouping-backslash">\\</span></span></span><span class="org-block"><span class="string"><span class="regexp-grouping-construct">|</span></span></span><span class="org-block"><span class="string">js</span></span><span class="org-block"><span class="string"><span class="regexp-grouping-backslash">\\</span></span></span><span class="org-block"><span class="string"><span class="regexp-grouping-construct">|</span></span></span><span class="org-block"><span  </span><span class="org-block"><span class="builtin">:recursive</span></span><span class="org-block"> t
 </span><span class="org-block"><span class="builtin">:publishing-function</span></span><span class="org-block"> org-publish-attachment

</span>
   <span class="bold">*Note*</span> that <span class="org-verbatim">=:publishing-function=</span> is set to <span class="org-verbatim">=org-publish-attachment=</span>.

<span class="org-hide">*</span><span class="org-level-2">* The </span><span class="org-level-2"><span class="italic">/publish/</span></span><span class="org-level-2"> component</span>

   To publish all with one command, we add the <span class="italic">/publish/</span> component. For this
   component I usually drop the suffix and just use the basename of the
   project.

<span class="org-block-begin-line">#+begin_src emacs-lisp
</span><span class="org-block"> (</span><span class="org-block"><span class="string">"org"</span></span><span class="org-block"> </span><span class="org-block"><span class="builtin">:components</span></span><span class="org-block"> (</span><span class="org-block"><span class="string">"org-notes"</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"org-static"</span></span><span class="org-block">))
</span><span class="org-block-end-line">#+end_src
</span>
   Now <span class="org-verbatim">=M-x org-publish-project RET org RET=</span> publishes everything
   recursively to <span class="org-verbatim">=~/public_html/=</span>. Target directories are created, if they
   don't yet exist.

<span class="org-hide">*</span><span class="org-level-2">* Pooh - can we publish now?</span>

   The good message is <span class="bold">*yes, we can*</span>. Just one little hump. Since we've put the
   definition for our publishing components in the <span class="org-verbatim">=*scratch*=</span> buffer, again,
   make sure all the <span class="italic">/components/</span> are enclosed by the lines

<span class="org-block-begin-line">#+begin_src emacs-lisp
</span><span class="org-block">(</span><span class="org-block"><span class="keyword">require</span></span><span class="org-block"> '</span><span class="org-block"><span class="constant">ox-publish</span></span><span class="org-block">)
(</span><span class="org-block"><span class="keyword">setq</span></span><span class="org-block"> org-publish-project-alist
      '(

       </span><span class="org-block"><span class="comment-delimiter">;; </span></span><span class="org-block"><span class="comment">... all the components ...
</span></span><span class="org-block">
      ))
</span><span class="org-block-end-line">#+end_src
</span>
   Move to the end of the first line and press <span class="org-verbatim">=C-x C-e=</span> to load
   <span class="org-verbatim">=org-publish=</span>. Now go to the end of the last line and press <span class="org-verbatim">=C-x C-e=</span>
   again. Repeat the last step after every change to your
   <span class="org-verbatim">=org-publish-project-alist=</span>.

   To publish your Org-mode files just type
   <span class="org-verbatim">=M-x org-publish-project RET org RET=</span> or use one of the shortcuts listed in
   the manual. If nothing went wrong, you should now be able to point your
   browser to <span class="org-link"><a href="http://localhost/~user/">http://localhost/~user/</a></span>, if <span class="org-verbatim">=mod_userdir=</span> is set up. If
   not, simply navigate to <span class="org-link"><a href="file:///home/user/public_html">file:///home/user/public_html</a></span> (you might use
   <span class="italic">/file -&gt; open/</span> from the file menu of your browser.

<span class="org-level-1">* Adding directories</span>

  As we add more and more files to <span class="org-verbatim">=~/org/=</span>, we will soon end up with filenames
  like '<span class="org-verbatim">=networking-ssh-sshd-config.org=</span>' or longer. What we need is a
  directory structure:

  <span class="org-code">: ~/org/
</span>  <span class="org-code">:   |- css/
</span>  <span class="org-code">:   |  `- stylesheet.css
</span>  <span class="org-code">:   |- Emacs
</span>  <span class="org-code">:   |  |- index.org
</span>  <span class="org-code">:   |  |- gnus.org
</span>  <span class="org-code">:   |  |- org.org
</span>  <span class="org-code">:   |  `- snippets.org
</span>  <span class="org-code">:   |- img/
</span>  <span class="org-code">:   |- index.org
</span>  <span class="org-code">:   `- remember.org
</span>
  If we hadn't added
  <span class="org-code">:      :recursive t
</span>  in the <span class="italic">/notes/</span> and <span class="italic">/static/</span> components already, we would have to do it now at
  the latest to export the subdirectories too.

<span class="org-level-1">* Overwrite the defaults</span>

  The defaults set by <span class="org-verbatim">=org-publish-project-alist=</span> may be overwritten. You might
  want to justify the export properties for single files. Be it the level of
  headlines, include extry scripts or different stylesheets. Org offers ways to adjust
  the settings for a single file.

<span class="org-hide">*</span><span class="org-level-2">* The export options template</span>

   The first choice is the <span class="italic">/export options template/</span> on top of the file. When in
   an Org-mode file, you may insert basic information using <span class="org-verbatim">=C-c C-e #=</span>
   (<span class="org-verbatim">=org-export-dispatch=</span>) plus "template".  You will be prompted for a template
   choice.  "default" will provide a template for common options, and "html"
   will provide a template for HTML-specific options.

   WARNING: Do <span class="bold">*not*</span> copy lines from the sample output below into your
   files. The template might change from release to release. Instead,
   insert a template as above and delete any entries that are not
   applicable.

   The default option inserts the following lines:
   
   <span class="org-code">: #+TITLE: filename with the extension omitted
</span>   <span class="org-code">: #+DATE: &lt;2013-06-04 Tue&gt;
</span>   <span class="org-code">: #+AUTHOR: Your name
</span>   <span class="org-code">: #+EMAIL: Your email address
</span>   <span class="org-code">: #+OPTIONS: ':t *:t -:t ::t &lt;:t H:3 \n:nil ^:t arch:headline author:t c:nil
</span>   <span class="org-code">: #+OPTIONS: creator:comment d:(not LOGBOOK) date:t e:t email:nil f:t inline:t
</span>   <span class="org-code">: #+OPTIONS: num:t p:nil pri:nil stat:t tags:t tasks:t tex:t timestamp:t toc:t
</span>   <span class="org-code">: #+OPTIONS: todo:t |:t
</span>   <span class="org-code">: #+CREATOR: Emacs 24.3.50.3 (Org mode 8.0.3)
</span>   <span class="org-code">: #+DESCRIPTION:
</span>   <span class="org-code">: #+EXCLUDE_TAGS: noexport
</span>   <span class="org-code">: #+KEYWORDS:
</span>   <span class="org-code">: #+LANGUAGE: en
</span>   <span class="org-code">: #+SELECT_TAGS: export
</span>

<span class="org-level-1">* Tired of export templates?</span>

 If you're like me, you will soon get tired of adding the same export options
 template to numerous files and adjust the title and paths in it. Luckily,
 Org-mode supports laziness and offers an additional way to set up files. All
 we need is a directory (e.g. <span class="org-verbatim">=~/.emacs.d/org-templates/=</span>) and create the
 following files there:

 + <span class="org-verbatim">=level-0.org=</span> \\
   This file contains all export options lines. The <span class="org-link">special comment section</span>
   will not work for files in subdirectories. Hence we always use the export
   options line
   <span class="org-special-keyword">:#+STYLE:</span> <span class="org-property-value">&lt;link rel="stylesheet" type="text/css" href="stylesheet.css" /&gt;</span>
   ...suitable for each file in the projects root folder
   (<span class="org-verbatim">=~/org/=</span> or <span class="org-verbatim">=~/B/=</span> in the examples). Just drop the <span class="org-verbatim">=#+TITLE=</span> since this
   will be different for every file and automatically set on export (based on
   the filename if omitted).
 + <span class="org-verbatim">=level-1.org=</span> \\
   This file contains all export options lines for the stylesheet suitable for
   each file in a subfolder of the projects root folder (e.g. <span class="org-verbatim">=~/org/emacs/=</span>
   or <span class="org-verbatim">=~/org/networking/=</span>). Just drop the <span class="org-verbatim">=#+TITLE=</span> again. The options line
   for the stylesheet looks like this:
   <span class="org-special-keyword">:#+STYLE:</span> <span class="org-property-value">&lt;link rel="stylesheet" type="text/css" href="../stylesheet.css" /&gt;</span>

 + Add more files for more levels.

 Now remove the <span class="org-link">special comment section</span> from the end of your Org-mode files in
 the project folders and change the export options template to

 <span class="org-code">: #+SETUPFILE: ~/.emacs.d/org-templates/level-N.org
</span> <span class="org-code">: #+TITLE: My Title
</span>
 Replace <span class="org-verbatim">=N=</span> with distance to the root folder (<span class="org-verbatim">=0=</span>, <span class="org-verbatim">=1=</span> etc.) of your project
 and press <span class="org-verbatim">=C-c=</span> twice while still on this line to apply the
 changes. Subsequent lines still overwrite the settings for just this one file.


<span class="org-hide">*</span><span class="org-level-2">* More level files</span>

  Also, these <span class="italic">/level-N/</span> files give us the chance to easily switch between different
  export setups. As an example, we could have a separate stylesheet and
  <span class="org-verbatim">=org-info.js=</span> setup for presentations, and put the appropriate options in a
  file named <span class="org-verbatim">=level-0-slides.org=</span>:

  <span class="org-code">: #+INFOJS_OPT: path:org-info.js
</span>  <span class="org-code">: #+INFOJS_OPT: toc:nil view:slide
</span>  <span class="org-code">: #+STYLE: &lt;link rel="stylesheet" type="text/css" href="slides.css" /&gt;
</span>
  Now it's as simple as typing '<span class="italic">/-slides/</span>' to change the appearance of any file
  in our project.

<span class="org-level-1">* More Projects</span>

   As we get used to note taking in org, we might add an <span class="org-verbatim">=org=</span> directory to most
   of our projects. All those projects are published as well. Project '<span class="org-verbatim">=~/B/=</span>'

   '<span class="org-verbatim">=~/public_html/C/=</span>', and so on. This leads to the problem of common

<span class="org-level-1">* Overview</span>

  Once there are lots of files and subdirectories, we're in the need of ways to
  easily navigate our notes in a browser. What we need now, is an index, an
  overview of all our note files.

<span class="org-hide">*</span><span class="org-level-2">* The sitemap</span>

   Org-modes great publishing also generates a recursive sitemap. Its name 
   defaults to <span class="org-verbatim">=sitemap.org=</span>, which get's in our way, since we have a real 
   startpage as <span class="org-verbatim">=sitemap.html=</span> <span class="org-footnote">[fn:5]</span>. Fortunately there is a configuration 
   option to change the name of the generated sitemap. To generate the sitemap,
   add these lines to the <span class="italic">/notes/</span> component:

<span class="org-block-begin-line">#+begin_src emacs-lisp
</span><span class="org-block"> </span><span class="org-block"><span class="builtin">:auto-sitemap</span></span><span class="org-block"> t                </span><span class="org-block"><span class="comment-delimiter">; </span></span><span class="org-block"><span class="comment">Generate sitemap.org automagically...
</span></span><span class="org-block"> </span><span class="org-block"><span class="builtin">:sitemap-filename</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"sitemap.org"</span></span><span class="org-block">  </span><span class="org-block"><span class="comment-delimiter">; </span></span><span class="org-block"><span class="comment">... call it sitemap.org (it's the default)...
</span></span><span class="org-block"> </span><span class="org-block"><span class="builtin">:sitemap-title</span></span><span class="org-block"> </span><span class="org-block"><span class="string">"Sitemap"</span></span><span class="org-block">         </span><span class="org-block"><span class="comment-delimiter">; </span></span><span class="org-block"><span class="comment">... with title 'Sitemap'.
</span></span><span class="org-block-end-line">#+end_src
</span>
   The sitemap will reflect the tree structure of the project. To access the
   sitemap easily, we could do two things:

   1. Setup the '<span class="italic">/UP/</span>' link of the Startpage to link to <span class="org-verbatim">=sitemap.html=</span> (see next
      section),
   2. use the '<span class="org-verbatim">=#+INCLUDE: sitemap.org=</span>' directive. Most of my Org-mode files
      contain a chapter called "<span class="italic">/Links/</span>" at the end of the file, which contains
      a subsection <span class="italic">/Sitemap/</span> that in turn just consists of that
      diretive. For the <span class="org-verbatim">=index.org=</span> files in the root directory, I include the
      sitemap as the first section.

   You can also change the position of folders with <span class="org-verbatim">=:sitemap-sort-folders=</span>,
   this can be set to <span class="org-verbatim">=last=</span> or <span class="org-verbatim">=first=</span> (default), to display folders last or 
   first.

<span class="org-hide">*</span><span class="org-level-2">* org-info.js</span>

   Another way to get additional links to navigate the structure is
   <span class="org-link"><a href="file:../code/org-info-js/index.org">org-info.js</a></span>. Let's set it up like this (either in every file, or in
   <span class="org-verbatim">=org-level-N.org=</span>, where <span class="org-verbatim">=N &gt; 0=</span>):

   <span class="org-code">: #+LINK_UP: index.html
</span>
   This makes the little <span class="italic">/UP/</span> link ('<span class="org-verbatim">=h=</span>') point to the <span class="org-verbatim">=index.html=</span> in the
   current directory.

   The <span class="org-verbatim">=index.org=</span> in the root of the project has the <span class="italic">/index file/</span> as section 2
   (which I may reach pressing '<span class="org-verbatim">=n=</span>' then), and the same option set like this:

   <span class="org-code">: #+LINK_UP: sitemap.html
</span>
   For an <span class="org-verbatim">=index.org=</span> in a subdirectory:

   <span class="org-code">: #+LINK_UP: ../index.html
</span>
   The <span class="org-verbatim">=LINK_HOME=</span> always points to the same file:

   <span class="org-code">: #+LINK_HOME: <a href="http://localhost/~user/index.html">http://localhost/~user/index.html</a>
</span>
   Please consider replacing the last one with a relative path (which will be
   different for every level of subdirectories).

   No matter where we are, we may always press <span class="org-verbatim">=H n=</span> and we face the sitemap.
   No matter where we are, we may always press <span class="org-verbatim">=h=</span> to move up the tree.
